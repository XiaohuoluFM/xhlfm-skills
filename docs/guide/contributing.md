# 共建方式

## 适合贡献什么

- 一个能在真实播客流程里反复使用的 Skill
- 一套明确的输入输出约定
- 一组帮助别人快速上手的示例
- 对某个工作环节的流程优化经验

## 推荐贡献步骤

1. 先确定你想解决的播客问题足够具体，例如“生成访谈提纲”或“把单集内容改写成平台文案”。
2. 在 `skills/` 下创建独立目录，并编写 `SKILL.md`。
3. 补充最少一个输入示例和一个理想输出示例。
4. 在文档站中登记或更新对应说明，帮助后来者找到它。
5. 提交 PR，并说明这个 Skill 的使用场景、边界和你最看重的效果标准。

## 一个高质量 Skill 的判断标准

- 能解决明确问题，而不是泛泛而谈
- 输入和输出清楚，别人可以直接复用
- 有边界说明，不会被误用
- 真实节省时间，或明显提升交付质量
- 适合在团队协作里接力使用

## 写作建议

- 尽量少写空泛描述，多写真实场景
- 多给例子，少给模糊判断
- 如果一个流程依赖特定工具或模型，写清楚前提
- 如果一个 Skill 还不稳定，也可以先标注为实验性

## 贡献时别忘了

- 保持命名清晰，目录结构稳定
- 避免把私有数据、私密链接或敏感信息提交进仓库
- 优先补充能帮助别人快速理解的上下文，而不是只给最终结论
