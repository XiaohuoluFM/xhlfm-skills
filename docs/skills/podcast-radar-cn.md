# 中文播客雷达

> Status: Available。已经实现，可安装使用。

## 这个 Skill 是做什么的

`中文播客雷达` 用来帮助你快速发现当下最值得关注的中文播客与热门单集。

它会基于《中文播客榜》的四类接口：

- 热门节目
- 热门播客
- 新锐节目
- 新锐播客

再把榜单数据整理成更适合真实使用的结果。

这里有一个很重要的中文语义约定：

- 当你说“最近热门播客”“推荐几个热门播客”时，它默认会优先返回最近值得点开的`单集 / 节目`
- 只有当你明确说“播客频道”“播客栏目”“节目主页”“频道级榜单”时，它才会优先返回`播客频道 / 栏目`

你可以把它理解成一个中文播客世界里的观察台：

- 你想知道最近大家都在听什么，它帮你看
- 你想研究别人的节目为什么热，它帮你拆
- 你想做推荐、策展或二次分发，它帮你先把线索捞出来

## 它特别适合做什么

### 1. 听众发现

- 找最近值得点开的中文播客单集
- 找某个分类下最近更热的节目
- 从热门榜和新锐榜里各挑一些值得入坑的内容

### 2. 创作者研究

- 找值得对标的热门节目
- 看同类播客最近什么题材更容易热
- 从标题、嘉宾和更新节奏里找可借鉴的线索

### 3. 策展与分发

- 为社群、Newsletter、公众号整理推荐单
- 找适合继续推荐、摘录、做卡片的热门单集
- 把“榜单”变成“可传播的发现结果”

## 适合谁用

- 想快速找最近值得听的中文播客的听众
- 想研究同类热门节目的播客创作者
- 想整理播客推荐清单的自媒体创作者
- 想寻找可摘录、可切片内容源的二创分发者

## 它怎么理解“播客”

这是这个 Skill 最刻意优化的一点。

在中文里，很多人说“给我几个最近热门播客”，其实真正想要的是：

- 最近哪几期最值得点开
- 最近哪几个节目最热
- 最近哪几条内容最适合听

所以默认情况下：

- `热门播客`
- `最近热门播客`
- `推荐几个最近很火的播客`

都会优先返回`单集 / 节目`。

如果你真正想看的是频道或栏目级对象，最好这样说：

- 给我最近值得关注的`播客频道`
- 帮我找几个热门的`播客栏目`
- 我想看`频道级`的热门榜单
- 给我一份值得关注的播客`节目主页`清单

### 面向听众

示例：

- 最近最热门的中文播客有哪些？
- 给我 10 个最近值得听的中文播客，偏商业和社会文化
- 我想找最近热起来的新锐中文播客
- 给我最近最热的播客单集，适合通勤听

预期输出：

- 热门单集或节目推荐清单
- 分类和热度信号
- 一句话推荐理由
- 小宇宙或 Apple 链接

### 面向播客创作者

示例：

- 我做科技播客，帮我找最近值得对标的中文播客频道
- 看看最近热门单集里，哪些标题和选题值得研究
- 给我一份最近新锐播客栏目里的潜在合作观察名单

预期输出：

- 对标频道 / 栏目列表
- 热门样本单集
- 主播/机构信息
- 可借鉴点

### 面向自媒体创作者

示例：

- 帮我整理一份最近适合推荐给职场人的中文播客清单
- 给我一份本周值得转发的热门播客单集摘要

预期输出：

- 按人群筛选后的节目 / 单集清单
- 推荐说明
- 可点击的节目入口

### 面向二创分发者

示例：

- 找 5 期最近适合做播客切片的热门单集
- 帮我挑出最近热度高、适合继续加工分发的中文播客单集

预期输出：

- 热门单集列表
- 推荐理由
- 可能适合二创的内容角度
- 原始链接

## 它返回的结果更像什么

`中文播客雷达` 的目标不是把 JSON 倒给你，而是把榜单变成一份你能直接用的工作结果。

典型输出会更像下面几种：

- 听众发现清单
- 创作者对标清单
- 自媒体策展清单
- 分发线索清单

例如，一次“听众发现”结果可能更接近这样：

```md
本周如果你想听中文播客里讨论度高、现在就值得点开的内容，可以先从这 5 期开始：

1. S8E9 鲁豫对话李睿｜我只是一名记者，留在了战争发生的地方
   岩中花述 / 艺术 / 高热度
   适合喜欢人物访谈、战争现场和女性叙事的听众。

2. 心流到来时，世界“暂时消失”
   不合时宜 / 社会与文化 / 热度与新鲜度兼具
   适合想听心理体验与自我观察主题的人。
```

一次“创作者对标”结果则更可能像：

```md
如果你在做商业访谈播客，这 6 档节目最值得先研究：

1. 知行小酒馆
   值得研究点：主题切口稳、标题清楚、长期保持更新。

2. 无人知晓
   值得研究点：话题密度高，评论互动强，更适合观察深度内容如何形成讨论。
```

## 推荐的提问方式

如果你只说“给我热门播客”，它当然能工作。

而且现在它会优先按中文口语习惯，把这句话理解成“给我最近值得点开的节目 / 单集”。

但如果你愿意多给一个目标，它会明显更聪明：

- 给我最近最热门的中文播客
- 给我最近最热门、偏社会文化的中文播客
- 给我最近最热门、偏社会文化、适合推荐给职场人听的中文播客

如果你想看的是频道或栏目，请直接说清楚：

- 给我最近值得关注的播客频道
- 给我 10 个热门播客栏目
- 给我一份适合研究的中文播客频道清单

你也可以直接在问题里说明用途：

- 我是听众，想找最近值得听的节目
- 我是创作者，想找能研究的对标对象
- 我在做内容策展，想整理一份播客推荐单
- 我想找适合继续分发的热门单集

## 什么时候它最好用

这个 Skill 最擅长的是：

- 找“当下值得关注什么”
- 找“某个方向里最近谁在热”
- 找“榜单里哪些内容最适合拿来继续整理和传播”

它不适合替代：

- 实时全网热搜
- 长周期的严肃行业统计
- 全平台、全口径的绝对排名

## 使用边界

你可以把它理解成“基于最近一轮榜单的中文播客发现器”。

需要留意：

- 这是基于最新一轮榜单的发现工具，不是分钟级实时热搜
- 数据不是全平台口径
- 新锐榜的分类字段缺失较多
- 某些榜单字段更适合作为内部参考信号，不适合直接当作结论展示
