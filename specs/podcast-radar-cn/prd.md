# 中文播客雷达 Skill PRD

- Status: Implemented v0
- Updated: 2026-04-01
- Proposed skill id: `podcast-radar-cn`
- Proposed display name: `中文播客雷达`

## 1. 背景

小火炉播客的第一个公开 Skill，希望先解决一个足够明确、需求也足够普遍的问题：

让用户快速发现当下值得关注的中文播客与单集，而不是手动刷榜、跳平台、逐个点开筛选。

这个 Skill 将基于《中文播客榜》的公开 API，把榜单数据转成更适合“发现、筛选、研究、分发”的结果，而不只是把榜单原样打印出来。

## 2. 为什么现在做

围绕“热门中文播客”的需求，在几个角色里都是真实存在的：

- 播客听众想快速找到最近最值得听的节目
- 播客创作者想知道同类节目里谁在涨、什么选题在热
- 自媒体创作者想找到值得引用、整理和推荐的内容源
- 做播客二创分发的人想找到适合摘录、做卡片、做切片的节目

我们在 2026 年 4 月 1 日实测观察到：

- 热门节目接口当前总量约 `814`
- 热门播客接口当前总量约 `6984`
- 新锐节目接口当前总量约 `58`
- 新锐播客接口当前总量约 `218`

这说明数据面足以支持发现类 Skill 的第一版，但也意味着 Skill 需要做分页、筛选、富化和输出结构化结论，而不是只取前 10 条。

## 3. 目标用户

### 3.1 播客听众

需求：

- 快速获取最近最热的中文播客
- 按类型、主题、更新新鲜度筛选
- 找到适合入坑的节目和单集

### 3.2 播客创作者

需求：

- 研究同类型节目现在谁在热
- 观察标题、选题、嘉宾和更新节奏
- 找到可对标节目与潜在合作对象

### 3.3 自媒体创作者

需求：

- 找到最近值得写推荐、整理合集的播客
- 快速提取节目简介、shownotes、平台链接
- 根据目标受众做播客策展与分发

### 3.4 播客二创分发者

需求：

- 寻找最近有传播潜力的热门单集
- 获取适合做卡片、摘要、摘录、切片的原始素材线索
- 根据节目描述和时间戳找到可二次加工的内容点

## 4. 核心问题定义

当前“找热门播客”这件事存在几个断点：

1. 榜单数据和详情信息分散在不同页面
2. 榜单只能看排名，不能直接回答“为什么值得关注”
3. 用户通常关心的是“适合我”的节目，而不是单纯的全榜前十
4. 创作者与分发用户需要的不只是节目名，还需要主题、链接与可行动结论

## 5. 产品目标

第一版 Skill 要做到：

1. 能从热门/新锐播客与热门/新锐单集四类榜单中检索数据
2. 能根据用户目标输出“发现结果”而不是原始 JSON
3. 能对结果做基础筛选：榜单类型、数量、分类、更新新鲜度
4. 能在必要时补充少量节目简介、单集描述与平台链接等上下文
5. 能面向不同角色生成不同风格的结果：
   - 听众发现清单
   - 创作者对标清单
   - 策展与分发线索清单

## 6. 非目标

第一版先不做：

- 跨平台全网热度聚合
- 精准的长期趋势预测
- 自动生成完整二创文案成品
- 自动下载音频、转录音频或抓取私有内容
- 对榜单做复杂统计建模或机器学习打分

## 7. 核心使用场景

### 场景 A：听众快速发现

用户说：

- 最近最热门的中文播客有哪些？
- 给我 10 个最近值得听的中文播客，偏商业和访谈
- 我想找最近热起来的新锐中文播客

Skill 输出：

- 排名清单
- 每档节目一句推荐理由
- 分类、更新新鲜度、入口链接

### 场景 B：创作者做对标研究

用户说：

- 我做商业访谈播客，帮我看看最近热门节目里哪些值得对标
- 给我看一下最近热播的科技播客和他们的代表单集
- 分析最近热门播客标题和选题有什么共同点

Skill 输出：

- 对标节目清单
- 分类、平均播放、互动、更新频率等信号
- 热门单集样本
- 初步观察结论

### 场景 C：策展与分发线索发现

用户说：

- 帮我整理一份最近值得推荐给职场人听的中文播客
- 最近哪些热门单集最适合做切片和摘要卡片？
- 找 5 期最近热度高、shownotes 完整、适合做二创分发的节目

Skill 输出：

- 适合推荐或继续加工的节目/单集清单
- 推荐理由
- 入口链接
- 可用于后续策展或二创的内容角度

## 8. 第一版能力清单

### 8.1 榜单检索

## 9. 数据来源与边界

### 9.1 主数据源

- 热门节目：`https://xyzrank.com/api/episodes`
- 热门播客：`https://xyzrank.com/api/podcasts`
- 新锐节目：`https://xyzrank.com/api/new-episodes`
- 新锐播客：`https://xyzrank.com/api/new-podcasts`

### 9.2 富化来源

- 小宇宙单集页：`https://www.xiaoyuzhoufm.com/episode/<eid>`
- 小宇宙播客页：`https://www.xiaoyuzhoufm.com/podcast/<pid>`

### 9.3 已确认的可用信息

我们实测确认：

- 榜单接口可返回排名、节目/播客名、分类、播放量、评论量、订阅量、链接等字段
- 小宇宙单集页 `__NEXT_DATA__` 中可以拿到 `description`、`shownotes`、`duration`、`playCount`、`commentCount`、`favoriteCount`、嵌套 `podcast` 信息
- 小宇宙播客页 `__NEXT_DATA__` 中可以拿到 `brief`、`description`、`subscriptionCount`、`episodeCount`、`podcasters`、`episodes`

### 9.4 产品边界与风险

- 《中文播客榜》本身是每周一更新，不是实时分钟级热榜
- 数据来源不是全平台，只覆盖小宇宙与 Apple Podcast 的一部分信号
- `primaryGenreName` 在新锐榜里缺失较多，不能完全依赖
- `openRate` / `avgOpenRate` 在实测数据中存在大于 `1` 的情况，不能按“概率”直接解释，应视为内部热度信号

## 10. 成功标准

第一版上线后，如果用户能稳定完成下面这些事情，就说明方向是对的：

- 3 分钟内拿到一份能直接点击收听的热门播客清单
- 创作者能快速得到一份可对标的节目列表
- 策展或二创用户能快速锁定值得继续加工的热门单集
- 输出结果不只是榜单罗列，而是带有上下文和行动建议

## 11. 命名方案

### 方案 A：中文播客雷达

优点：

- 好记
- 不只局限于“热榜”，也适合后续扩展到发现、对标、趋势
- 同时适合听众和创作者

### 方案 B：播客热榜发现器

优点：

- 功能表达直接

问题：

- 太像工具名，不像一个可以扩展的 Skill 名称

### 方案 C：中文播客趋势侦测

优点：

- 强调趋势

问题：

- 语感偏硬，不够友好

## 12. 推荐名称

- 用户侧名称：`中文播客雷达`
- 技术侧 skill id：`podcast-radar-cn`

原因：

- 既能覆盖“热榜发现”，也能覆盖“新锐发现”“创作者对标”“二创线索发现”
- 后续如果扩展到分类推荐、播客观察、趋势快报，也不需要改名
